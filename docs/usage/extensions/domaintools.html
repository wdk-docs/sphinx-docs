域工具¶

这个 Sphinx extension 提供了一个简单的 sphinx domain 创建工具。

安装¶

pip install sphinxcontrib-domaintools

用法¶

在这个例子中，为 GNU Make 创建了一个简单的域:

from sphinxcontrib.domaintools import custom_domain

def setup(app):
    app.add_domain(custom_domain('GnuMakeDomain',
        name  = 'make',
        label = "GNU Make",

        elements = dict(
            target = dict(
                objname      = "Make Target",
                indextemplate = "pair: %s; Make Target",
            ),
            var   = dict(
                objname = "Make Variable",
                indextemplate = "pair: %s; Make Variable"
            ),
        )))

您可以在 `sphinxcontrib.makedomain`_ 包中找到完整的示例。

您可以在 sphinxcontrib-cmakedomain 包中找到更复杂的示例。

参考¶

custom_domain(class_name, name='', label='', elements = {}):

创建自定义域。

对于每个给定元素，都创建了一个指令和一个引用和索引的角色。

参数

class_name – 您新域的ClassName(例如 GnuMakeDomain)

:param name : 域简称(指令的一部分，例如 make) :param label : 域长名(例如 GNU Make) :param elements :

域指令/角色的字典

元素值是包含以下可能条目的字典:

  • objname - 条目的长名称，默认为条目的密钥

  • role - 角色名称，默认为条目的密钥

  • indextemplate - e.g. pair: %s; Make Target, 其中％s将是您角色的匹配部分。 您可以将此保留为空，默认为 pair: %s; <objname>

  • parse_node - 带有签名 (env，sig，signode) 的函数，默认为 None 。

  • fields - 解析字段映射到的字段列表。这将作为 doc_field_types 参数传递给Domain。

  • ref_nodeclass - 作为 XRefRole 的 innernodeclass 传递的类，默认为 None 。
